In this lesson I learned about Network Address Translation or NAT. NAT is helpful because it supplies multiple private devices which share the same network with a single public IP address when connecting to the internet. This also hides private addresses from the public. 

## NAT

- Application-Level Gateway (ALG): A router feature that inspects certain protocols (such as SIP or FTP) and modifies traffic so it can pass through NAT, though it can sometimes cause problems.

- Double NAT: A networking issue that occurs when two devices on the same network are both performing NAT, often causing problems with gaming, port forwarding, and remote access.

- Dynamic NAT: A many-to-many mapping where private IPs are assigned a public IP from a pool on a temporary basis.

- NAT Table (Translation Table): A temporary log kept by a router that maps private IP addresses and port numbers to a public IP address so return traffic can reach the correct device.

- Network Address Translation (NAT): A method that lets many private devices share one public IP address while hiding their internal addresses from the internet.

- Peer-to-Peer (P2P): A type of network communication where two devices connect directly to each other without going through a central server.

- Port Address Translation (PAT): A many-to-one mapping where multiple private IPs share a single public IP, using port numbers to track connections. Also known as NAT Overload.

- Port Forwarding: A router configuration that directs incoming traffic on a specific port to a designated device inside the private network.

- Static NAT: A one-to-one mapping where a specific private IP is always translated to the same public IP.

- Voice over IP (VoIP): Technology that allows voice calls to be made over the internet instead of traditional phone networks.
