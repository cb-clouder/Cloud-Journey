In this lesson, I learned how to check for network traffic failure using the ping command on my computer's terminal. I pinged my IP address which showed that my computer was communicating with my wi-fi network. I pinged the IP address 8.8.8.8 to prove that my computer was communicating with the outside world. I also pinged a known domain (www.google.com) and received responses. This proved that the DNS server was responding.
I also learned how to troubleshoot using packet capture (pcap files) when communications fail. PCAP files provide a deeper view. They record the actual traffic sent and received by a device. By opening a pcap in a tool such as DynamiteLab, you can see whether requests left your computer, whether replies were received, and what errors occurred along the way.

- Default Gateway: The IP address of the router that connects a local network to external networks, providing the path to destinations outside the subnet.

- Domain Name System (DNS): A service that translates human-readable names, such as google.com, into numerical IP addresses so devices can communicate.
